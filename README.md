## 自己PR
私の強みは、抽象的な概念を言語化し、開発に落とし込む力です。

新規機能の開発においては、仕様をもとに概念を言語化し、PdMやEMと議論を重ねながら要件を具体化します。特に意識しているのは、概念の境界を明確にすることです。これにより、チーム内での認識齟齬を防ぎ、将来的な仕様変更にも柔軟に対応できる堅牢な設計が可能になります。

例えば、建設業向けシステムで「案件」と「受注」の関係を設計する際、「受注」という概念を社内で完結させるべきかどうかを議論しました。この境界設定が重要なのは、インボイス制度の適用時に発生し得る金額差異をどこまで許容するかに関わるためです。結果として、「受注」は社内完結の概念として統一され、将来的に社外連携が必要となった場合の設計方針まで議論を進めることができました。

私はこうした抽象的な課題を構造的に捉えて言語化し、チームで議論を行うプロセスそのものを楽しんでいます。今後もこの強みを生かし、顧客の課題解決とプロダクトの成長に貢献していきたいと考えています。

## この先やってみたいこと

### 熱狂的な事業への挑戦
私は、組織全体に熱量のある事業に挑戦しているときに最も喜びを感じます。

具体的には組織が「こと」に向かい、事業が伸び、そして新たな挑戦の機会が創出される。このサイクルが回る環境の中でこそ、自身の視座やスキルをより高めることができると感じています。こうした変化と挑戦に満ちた事業フェーズで、技術を通じて推進力となることを目指しています。

### バックエンドへ軸足に置いた開発
私は今後、バックエンド領域を専門として技術力を高めていきたいと考えています。

日々の学習や開発の中で興味が自然とバックエンドに向かっており、パフォーマンス改善やデータモデリング、スケーラブルな設計といった領域に強い関心を持っています。これらの技術的挑戦を通じて、堅牢で保守性の高いシステムを支える基盤をつくれるエンジニアを目指します。

### プロダクト思考の開発
私は、抽象的な課題を構造的に捉えて解決することが好きです。

技術を磨くだけでなく、顧客の課題を深く理解し、その本質に基づいて設計・実装を行う「プロダクト思考の開発」に魅力を感じています。技術的なパターンや解法にとらわれず、常に「なぜそれを作るのか」という視点から課題を言語化し、価値のある機能を生み出す開発を続けていきたいと考えています。
